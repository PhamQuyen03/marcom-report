Giải pháp AKS cho app Next.js và report HTML theo mô hình only slug

Hướng này giữ nguyên app Next.js, login, docs và report portal hiện tại. Report HTML vẫn nằm trong public/reports, còn public endpoint của report chỉ còn dạng /reports/<slug>.

Vì sao AKS là hướng phù hợp

Repo hiện tại

Đây là app Next.js, không phải static-only.

Platform công ty

Đã có AKS, ACR, Azure DevOps và team DevOps vận hành.

Một web portal duy nhất

App tự tích hợp Microsoft Entra External ID cho cả internal và external users.

Mô hình slug-only

Source

/reports/nua-dau-nam-t1-t5-2026

App sẽ map slug route sang file HTML thực tương ứng trong public/reports.

  • Deploy AKS, smoke check và notify Teams.

Public entry và publish layer

Lớp này phụ trách nhận request từ domain, áp WAF và route request vào app thông qua publish layer của AKS.

Thành phần được chốt là Azure Front Door Premium ở lớp public entry và AKS application routing Gateway API implementation với gatewayClassName: approuting-istio làm publish layer trong cluster.

Thứ tự truy cập khuyến nghị

  • Internal Users / External Users -> Azure Front Door Premium.
  • Azure Front Door Premium -> Gateway API.
  • Gateway API -> AKS Next.js App.
  • AKS Next.js App -> Microsoft Entra External ID cho OIDC login và callback.

Khi nói Ingress layer trong tài liệu này thì nên hiểu là Azure Front Door Premium ở bên ngoài và Gateway API ở trong cluster. Phần login và authorization nằm trong app.

Identity và authorization model

Identity

Microsoft Entra External ID là identity provider cho portal. Internal user và external user cùng đi qua flow mà app cấu hình với External ID.

Authorization

App kiểm tra quyền theo path và slug, dùng role mapping hoặc ACL trong database / metadata store.

SSL trên AKS

Hướng 1: cert-manager

Tự động cấp và renew certificate nếu platform đang dùng Let's Encrypt.

Hướng 2: certificate nội bộ

Certificate được lưu vào Kubernetes Secret và Gateway hoặc lớp publish dùng để terminate TLS.

Flow pipeline chuẩn

  1. Checkout code.
  2. Install dependency bằng pnpm.
  3. Chạy lint.
  4. Validate report slug.
  5. Build Next.js.
  6. Build Docker image.
  7. Push image lên ACR.
  8. Deploy lên AKS.
  9. Chờ rollout thành công.
  10. Smoke check /reports/<slug>.
  11. Notify Teams.

Rollback

Rollback theo chuẩn AKS: rollback deployment hoặc redeploy image tag cũ.

kubectl rollout undo deployment/marcom-report -n marcom-report

Lưu ý build

Để CI/CD production ổn định, current setup dùng:

  • next dev --turbopack cho local dev.
  • next build --webpack cho production build.
