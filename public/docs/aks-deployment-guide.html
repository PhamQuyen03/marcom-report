Giải pháp AKS cho app Next.js và report HTML theo mô hình only slug

Hướng này giữ nguyên app Next.js, login, docs và report portal hiện tại. Report HTML vẫn nằm trong public/reports, còn public endpoint của report chỉ còn dạng /reports/<slug>.

Vì sao AKS là hướng phù hợp

Repo hiện tại

Đây là app Next.js, không phải static-only.

Platform công ty

Đã có AKS, ACR, lớp publish traffic của cluster và team DevOps vận hành.

CI/CD

Có thể bám theo chuẩn release enterprise chung của công ty.

Mô hình slug-only

Ingress layer

Lớp này phụ trách expose domain, route request vào app Next.js và terminate TLS.

Trong hướng này, lựa chọn được chốt là AKS application routing Gateway API implementation với gatewayClassName: approuting-istio.

Ingress layer khuyến nghị

  • Dùng AKS application routing Gateway API implementation.
  • Dùng gatewayClassName: approuting-istio.
  • Dùng Gateway + HTTPRoute cho publish traffic.

Khi nói Ingress layer trong tài liệu này thì nên hiểu là lớp Gateway API của AKS application routing.

SSL trên AKS

Hướng 1: cert-manager

Tự động cấp và renew certificate nếu platform đang dùng Let's Encrypt.

Hướng 2: certificate nội bộ

Certificate được lưu vào Kubernetes Secret và ingress layer dùng để terminate TLS.

Flow pipeline chuẩn

  1. Checkout code.
  2. Install dependency bằng pnpm.
  3. Chạy lint.
  4. Validate report slug.
  5. Build Next.js.
  6. Build Docker image.
  7. Push image lên ACR.
  8. Deploy lên AKS.
  9. Chờ rollout thành công.
  10. Smoke check /reports/<slug>.
  11. Notify Teams.

Rollback

Rollback theo chuẩn AKS: rollback deployment hoặc redeploy image tag cũ.

kubectl rollout undo deployment/marcom-report -n marcom-report
