Giải pháp AKS / Azure DevOps

CI/CD report HTML trên AKS

Hướng này phù hợp khi công ty đã có sẵn platform Kubernetes, team DevOps vận hành, ACR, Ingress và quy trình release chuẩn qua Azure DevOps.

Chuẩn enterprise

Release đi theo image tag, Ingress, TLS và rollout chuẩn Kubernetes.

Rollback

Dễ rollback bằng image tag hoặc rollout undo.

SSL

Thường xử lý qua Ingress, cert-manager hoặc secret nội bộ.

Đánh đổi

Phức tạp hơn VM nếu chỉ xét riêng bài toán static HTML.

Khi nào nên chọn

Nên chọn khi

  • Công ty đã có sẵn platform Kubernetes.
  • Đã có ACR, Ingress, cert-manager, logging và monitoring.
  • Team DevOps quen vận hành workload trên AKS.
  • Muốn đưa bài toán report HTML vào cùng chuẩn platform với service khác.

Không nên chọn khi

  • Chỉ cần publish static HTML đơn giản.
  • Chưa có cluster và chưa có team vận hành.
  • Ưu tiên chi phí thấp và độ đơn giản tuyệt đối.
Kết luận ngắn: nếu platform Kubernetes và team DevOps đã có sẵn, AKS là một hướng hợp lý và chuẩn enterprise cho CI/CD của report HTML.

Hai mô hình triển khai phù hợp

Mô hình A: đóng report vào image Nginx

  • Azure DevOps gom report vào image nginx.
  • Build image.
  • Push lên ACR.
  • Deploy lên AKS.

Ưu điểm: rollback tốt, audit rõ, dễ kiểm soát.

Nhược điểm: đổi 1 file HTML cũng phải build image mới.

Mô hình B: HTML nằm ngoài image

  • Report được copy lên Azure Files hoặc Blob.
  • Pod Nginx mount storage vào web root.
  • Ingress publish ra domain.

Ưu điểm: update nội dung nhanh, không cần build image mỗi lần.

Nhược điểm: phức tạp hơn về storage và permission.

Khuyến nghị: nếu mới bắt đầu với AKS cho use case này, nên đi từ mô hình A trước.

SSL trên AKS

Hướng 1: cert-manager

  • Tự động cấp và renew.
  • Chuẩn Kubernetes.
  • Phù hợp khi dùng Let's Encrypt.

Hướng 2: certificate nội bộ

  • Phù hợp policy doanh nghiệp.
  • Certificate được đưa vào Kubernetes Secret.
  • Ingress dùng secret để terminate TLS.
Khác với VM: trên AKS thường không SSH vào node để chạy certbot. SSL được xử lý ở layer Ingress hoặc secret Kubernetes.

Dockerfile và Nginx config

Dockerfile mẫu

FROM nginx:alpine

COPY publish/ /usr/share/nginx/html/
COPY nginx.conf /etc/nginx/conf.d/default.conf

Nginx config mẫu

server {
    listen 80;
    server_name _;

    root /usr/share/nginx/html;
    index index.html;

    location / {
        try_files $uri $uri/ $uri/index.html =404;
    }
}

Kubernetes manifest tối giản

Deployment

apiVersion: apps/v1

metadata:
  name: marcom-report

  replicas: 2

Service

apiVersion: v1
kind: Service
metadata:
  name: marcom-report
spec:
  ports:
    - port: 80

Ingress

apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: marcom-report
spec:
  tls:
    - secretName: reports-tls

Flow pipeline chuẩn

  1. checkout code
  2. validate cấu trúc report
  3. build publish tree
  4. build image và push ACR nếu dùng mô hình A
  5. deploy AKS bằng kubectl apply hoặc helm upgrade
  6. đợi rollout thành công
  7. smoke check URL public
  8. gửi notify Microsoft Teams

Smoke check và rollback

Smoke check

curl -I https://reports.example.com/nua-dau-nam-t1-t5-2026/v2/
curl -I https://reports.example.com/nua-dau-nam-t1-t5-2026/latest/
curl -I https://reports.example.com/report/bao-cao-doanh-thu-q2.html

Rollback mô hình A

kubectl rollout undo deployment/marcom-report -n marcom

Hoặc redeploy image tag cũ qua Helm hay manifest.

Teams notification

Nên gửi gì

  • tên pipeline
  • image tag hoặc release version
  • môi trường
  • URL public
  • trạng thái success hoặc failure
  • timestamp

Ví dụ payload

{
  "text": "Deploy AKS thành công cho marcom-report image 2026.06.23.1. URL: https://reports.example.com/"
}

  • Ingress hoạt động

Rủi ro chính

Rủi ro kỹ thuật

  • CI/CD phức tạp hơn VM.
  • Sai cấu hình Ingress hoặc TLS.
  • Rollback không rõ nếu không quản image tag chặt.

Cách giảm rủi ro

  • Chuẩn hóa chart hoặc manifest.
  • Dùng image tag rõ ràng.
  • Lưu immutable artifact.
  • Tách verify riêng cho DNS, TLS và path routing.

Kết luận

Nếu công ty đã có sẵn platform Kubernetes và team DevOps vận hành, thì AKS là một lựa chọn hợp lý cho bài toán report HTML này.

Khuyến nghị thực tế là bắt đầu bằng mô hình đóng report vào image Nginx, chuẩn hóa release qua ACR + AKS + Ingress, để pipeline xử lý latest, và để Azure DevOps xử lý notify Teams.

Tài liệu này là bản chi tiết riêng cho hướng triển khai AKS.
