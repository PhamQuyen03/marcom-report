Giải pháp 2 / VM với Nginx

Hướng triển khai chi tiết với VM và Nginx

Tài liệu này mô tả cách triển khai nhiều report HTML trên VM Linux dùng Nginx, hỗ trợ cả mô hình versioned folder và mô hình endpoint theo tên file .html, kèm rollback và notify Microsoft Teams.

Điểm mạnh

Rất linh hoạt, kiểm soát hoàn toàn file system và URL.

Chi phí

Thường thấp nhất về hạ tầng thuần nếu tự vận hành.

Rollback

Nhanh nhất nếu dùng symlink cho latest ở mô hình versioned.

Đánh đổi

Nặng công vận hành nhất trong các giải pháp.

Khi nào nên chọn

Nên chọn khi

  • Muốn tối ưu chi phí hạ tầng thuần.
  • Cần toàn quyền với file system, web root và routing.
  • Muốn hỗ trợ cả folder version lẫn file HTML public trực tiếp.
  • Muốn kiểm soát latest bằng symlink hoặc alias nếu cần.
  • Chấp nhận tự vận hành server.

Không nên chọn khi

  • Team không muốn vận hành Linux server.
  • Team không muốn tự xử lý SSL, backup, monitoring.
  • Muốn mọi thứ managed hoàn toàn.

Deployment flow

Repository / CI Pipeline
  -> Build hoặc package artifact
  -> Copy artifact lên VM qua SSH / SCP / rsync
  -> Extract hoặc copy vào release tree
  -> Validate file bắt buộc
  -> Update latest nếu đang dùng versioned
  -> Smoke check URL public tương ứng
  -> Notify Microsoft Teams
Lưu ý: latest chỉ là tùy chọn cho mô hình versioned. Với mô hình file-based, endpoint công khai có thể là chính tên file .html.

Phân tách rõ phần tạo tay và phần CI/CD tự động

Chỉ làm tay một lần lúc bootstrap

  • tạo VM
  • gắn domain hoặc cấu hình DNS
  • cài Nginx
  • cài SSL với certbot
  • tạo user deployer
  • tạo thư mục gốc như /var/www/reports
  • set permission ban đầu
  • cấu hình Nginx server block
  • chuẩn bị SSH key hoặc service connection từ Azure DevOps sang VM
  • tạo webhook Microsoft Teams và lưu vào secret

Azure DevOps sẽ tự động làm mỗi lần deploy

  • đọc source từ Git
  • build hoặc collect artifact
  • xác định endpoint cần publish theo folder hoặc theo file
  • copy folder report hoặc file HTML lên VM bằng scp, rsync hoặc SSH task
  • tạo folder version mới nếu chưa có
  • copy file HTML, CSS, JS, assets vào đúng path
  • kiểm tra index.html hoặc file HTML public tương ứng
  • cập nhật latest nếu đang dùng mô hình versioned
  • smoke check URL public tương ứng
  • gửi notify success hoặc failure về Microsoft Teams
Tóm lại: phần hạ tầng nền tạo tay một lần. Sau đó pipeline có thể tự copy folder report hoặc file HTML từ Git hay artifact lên VM và publish hoàn toàn tự động.

Khuyến nghị: dùng symlink cho latest nếu chọn mô hình versioned. Với mô hình file-based, endpoint public có thể là chính file .html.

Chuẩn bị hạ tầng

VM

  • Ubuntu LTS.
  • VM public hoặc đi qua load balancer.
  • SSH key-based access.
  • Không dùng password login nếu có thể.

Domain và DNS

  • Có quyền quản lý DNS.
  • A record trỏ về public IP của VM.
  • Hoặc CNAME nếu có lớp trung gian.

Nền tảng vận hành

  • Cài nginx.
  • Cài certbot.
  • Tạo user deploy riêng, ví dụ deployer.
  • Chuẩn bị webhook Microsoft Teams.

Cài đặt cơ bản

Cài Nginx

sudo apt update
sudo apt install -y nginx

Tạo release directory

sudo mkdir -p /var/www/reports
sudo chown -R deployer:deployer /var/www/reports
sudo chmod -R 755 /var/www/reports

Bước này chỉ làm một lần. Sau đó pipeline sẽ tự copy folder report mới hoặc file HTML mới vào bên dưới /var/www/reports/.

Kiểm tra Nginx

sudo nginx -t
sudo systemctl enable nginx
sudo systemctl restart nginx

Cấu hình Nginx đề xuất

server {
    listen 80;
    server_name example.com www.example.com;

    root /var/www/reports;
    index index.html;

    location / {
        try_files $uri $uri/ $uri/index.html =404;
    }

    location ~ /\. {
        deny all;
    }

    access_log /var/log/nginx/reports.access.log;
    error_log /var/log/nginx/reports.error.log;
}

Rule này hỗ trợ tốt cả URL dạng /slug/v1/, /slug/latest/ và file public như /report/bao-cao-doanh-thu-q2.html.

SSL với Let's Encrypt

sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d example.com -d www.example.com
sudo certbot renew --dry-run

Khi nào có thể tự cấp SSL

Nếu bạn SSH được vào VM và có đủ quyền sudo, thì thông thường bạn có thể tự:

  • cài certbot
  • lấy SSL từ Let's Encrypt
  • gắn SSL vào Nginx
  • bật và kiểm tra auto-renew

Điều kiện cần đủ

  • domain hoặc subdomain đã trỏ đúng về VM
  • cổng 80 và 443 đang mở
  • bạn có quyền sudo
  • Nginx đang chạy ổn

Flow thực tế

sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com -d www.your-domain.com
sudo certbot renew --dry-run
Lưu ý: nếu DNS chưa trỏ đúng về VM thì certbot sẽ không xác thực được domain. Trong đa số trường hợp, certbot tự tạo systemd timer để auto-renew, không cần tự cấu hình cron tay.

Bước 1-3

  • Build hoặc collect artifact.
  • Copy artifact lên VM bằng scp, rsync hoặc SSH task.
  • Extract hoặc move vào release tree.
rsync -avz artifact/ deployer@example.com:/var/www/releases/tmp/

mkdir -p /var/www/reports/nua-dau-nam-t1-t5-2026/v2
cp -R /var/www/releases/tmp/nua-dau-nam-t1-t5-2026/v2/* /var/www/reports/nua-dau-nam-t1-t5-2026/v2/

Nếu repo có cấu trúc như report/nua-dau-nam-t1-t5-2026/v1/, report/nua-dau-nam-t1-t5-2026/v2/ hoặc report/bao-cao-doanh-thu-q2.html thì Azure DevOps có thể tự copy cả folder lẫn file HTML lên VM.

Bước 4-7

  • Kiểm tra file bắt buộc.
  • Cập nhật latest nếu có.
  • Smoke check URL public tương ứng.
  • Notify Microsoft Teams khi success hoặc failure.
test -f /var/www/reports/nua-dau-nam-t1-t5-2026/v2/index.html
ln -sfn /var/www/reports/nua-dau-nam-t1-t5-2026/v2 /var/www/reports/nua-dau-nam-t1-t5-2026/latest

Rollback và notify

Rollback với mô hình versioned

Rollback tốt nhất là đổi lại latest, không xóa version mới ngay.

ln -sfn /var/www/reports/nua-dau-nam-t1-t5-2026/v1 /var/www/reports/nua-dau-nam-t1-t5-2026/latest

Sau đó test lại URL /latest/.

Rollback với file HTML trực tiếp

  • Nếu overwrite đúng một file public thì rollback là copy lại file cũ.
  • Nếu publish file mới bằng tên khác thì rollback là đổi link đang share.

Payload notify tối thiểu

  • tên report
  • version hoặc tên file endpoint
  • môi trường
  • URL public
  • URL latest nếu có
  • thời gian deploy
  • người hoặc pipeline thực hiện
  • trạng thái success hoặc failure

Ví dụ notify

{
  "text": "Deploy thành công report bao-cao-doanh-thu-q2.html. URL: https://domain.com/report/bao-cao-doanh-thu-q2.html"
}

Checklist triển khai

Checklist chuẩn bị

  • có VM Linux
  • có public IP hoặc lớp route phía trước
  • có quyền sửa DNS
  • có domain
  • có SSH key
  • cài Nginx
  • cài certbot
  • có user deploy riêng
  • có release directory
  • có webhook Microsoft Teams

Checklist go-live

  • DNS trỏ đúng
  • Nginx config pass nginx -t
  • SSL hoạt động
  • URL public mở được
  • URL latest mở được nếu có dùng
  • log access và error đang ghi bình thường
  • permission thư mục đúng
  • notify Teams gửi được ở cả success và failure

Checklist vận hành định kỳ

  • cập nhật security patch cho VM
  • kiểm tra disk usage
  • kiểm tra certificate expiry
  • kiểm tra log Nginx
  • kiểm tra backup
  • rà SSH access và user không còn dùng
  • kiểm tra webhook notify còn hoạt động

Rủi ro chính và cách giảm rủi ro

SSL hết hạn

  • Bật auto renew.
  • Có alert trước ngày hết hạn.

Permission sai

  • Chuẩn hóa owner và chmod.
  • Không deploy bằng nhiều user khác nhau.

latest trỏ sai

  • Chỉ update sau khi smoke check.
  • Ghi log version nào đang là latest.

Server đầy disk

  • Có cleanup policy cho artifact tạm.
  • Monitor disk định kỳ.

Mất dữ liệu do deploy lỗi

  • Không overwrite version cũ.
  • Rollback bằng symlink.
  • Backup trước thay đổi lớn.

Overwrite nhầm file HTML public

  • Giữ backup hoặc artifact của bản cũ.
  • Không deploy đè nếu chưa có naming rule rõ ràng.

Notify không gửi được

  • Tách bước notify thành step rõ ràng trong pipeline.
  • Log payload và HTTP status của webhook.
  • Có fallback alert nếu cần.

Mẫu lệnh tham khảo

Deploy versioned

REPORT_SLUG="nua-dau-nam-t1-t5-2026"
REPORT_VERSION="v2"
TARGET_DIR="/var/www/reports/${REPORT_SLUG}/${REPORT_VERSION}"
LATEST_LINK="/var/www/reports/${REPORT_SLUG}/latest"

mkdir -p "${TARGET_DIR}"
cp -R "./artifact/${REPORT_SLUG}/${REPORT_VERSION}/"* "${TARGET_DIR}/"
test -f "${TARGET_DIR}/index.html"
ln -sfn "${TARGET_DIR}" "${LATEST_LINK}"

Deploy file HTML trực tiếp

mkdir -p /var/www/reports/report
cp "./artifact/report/bao-cao-doanh-thu-q2.html" \
  "/var/www/reports/report/bao-cao-doanh-thu-q2.html"
test -f "/var/www/reports/report/bao-cao-doanh-thu-q2.html"

Smoke check

curl -I https://domain.com/nua-dau-nam-t1-t5-2026/v2/
curl -I https://domain.com/nua-dau-nam-t1-t5-2026/latest/
curl -I https://domain.com/report/bao-cao-doanh-thu-q2.html

Kết luận

VM + Nginx phù hợp nếu mục tiêu là chi phí hạ tầng thấp, URL linh hoạt, rollback nhanh và kiểm soát hoàn toàn file system.

Hướng này không bắt buộc mọi report phải có version. Bạn có thể vừa chạy mô hình versioned cho các report cần latest, vừa chạy mô hình file HTML public trực tiếp cho các báo cáo đơn lẻ.

Tài liệu này là bản chi tiết cho phần VM với Nginx trong bộ tài liệu kiến trúc CI/CD tổng quan.
