Giải pháp VM với Nginx theo mô hình only slug

Public endpoint của report chỉ còn một slug ổn định theo dạng /reports/<slug>. File thật vẫn là HTML và Nginx sẽ rewrite sang file tương ứng.

Mô hình URL và file system

Public route

https://domain/reports/nua-dau-nam-t1-t5-2026
https://domain/reports/bao-cao-ai-2026

File trên VM

/var/www/reports/
  reports/
    nua-dau-nam-t1-t5-2026.html
    bao-cao-ai-2026.html

Nginx config đề xuất

server {
    listen 80;
    server_name example.com www.example.com;

    root /var/www/reports;

    location /reports/ {
        try_files $uri $uri/ $uri.html =404;
    }
}

Request /reports/bao-cao-ai-2026 sẽ map sang file /var/www/reports/reports/bao-cao-ai-2026.html.

Bootstrap một lần

  • Tạo VM, DNS, Nginx, SSL, user deploy và webhook Teams.
  • Tạo thư mục /var/www/reports.
  • Chuẩn bị SSH key hoặc service connection từ Azure DevOps.

Pipeline tự động mỗi lần deploy

  1. Checkout source.
  2. Build hoặc lấy artifact của slug.
  3. Copy file HTML của slug lên VM.
  4. Smoke check /reports/<slug>.
  5. Notify Microsoft Teams.

SSL với Let's Encrypt

sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d example.com -d www.example.com
sudo certbot renew --dry-run

Nếu domain trỏ đúng về VM, cổng 80/443 mở và bạn có quyền sudo, thì có thể tự setup SSL và auto-renew.

Rollback

Với mô hình only slug, rollback chỉ là copy lại file HTML cũ của chính slug đó rồi reload Nginx nếu cần.
