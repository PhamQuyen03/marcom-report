Azure DevOps / CI/CD Architecture

Mô hình only slug cho report HTML

Public endpoint của mỗi report chỉ còn một slug ổn định theo dạng /reports/<slug>. Không dùng version path và không dùng latest. Rollback được xử lý bằng artifact cũ của chính slug đó.

Public URL

/reports/<slug>

Artifact

Một artifact rõ ràng cho mỗi slug.

Rollback

Redeploy artifact hoặc image cũ của cùng slug.

Notify

Success và failure đều gửi Teams.

Quy ước route

https://domain/reports/bao-cao-ai-2026

Source file:
public/reports/nua-dau-nam-t1-t5-2026.html
public/reports/bao-cao-ai-2026.html

System Architecture Diagram

Identity And Access Model

  • Microsoft Entra External ID là identity provider cho portal.
  • Web Portal tự xử lý login, callback, session và logout.
  • Authorization Store giữ role mapping và ACL theo report hoặc path.
  • System Architecture Diagram chỉ giữ mức logical high-level, không nhồi CI/CD hay auth flow chi tiết.

Nguyên tắc CI/CD chung

  • Build một lần, deploy một artifact rõ ràng cho từng slug.
  • Không build lại trong production stage.
  • Validate sớm file slug, route public và smoke check.
  • Rollback bằng artifact cũ của chính slug đó.
  • Success hoặc failure đều notify về Microsoft Teams.

Luồng pipeline chuẩn

  1. Detect thay đổi ở report source.
  2. Validate naming rule của slug.
  3. Build artifact hoặc build image.
  4. Deploy lên môi trường mục tiêu.
  5. Smoke check /reports/<slug>.
  6. Notify Microsoft Teams.
  7. Nếu cần rollback thì redeploy artifact cũ của slug đó.

Giải pháp: AKS

Đây là giải pháp phù hợp nhất với yêu cầu hiện tại khi repo chính vẫn là app Next.js, công ty đã có platform Kubernetes, ACR, Azure DevOps, team DevOps vận hành và portal cần phục vụ cả internal lẫn external users.

Logical access view

Internal Users / External Users -> Public Web Entry -> Web Portal

Checklist

  • Build image Next.js.
  • Push image lên ACR.
  • Deploy lên AKS.
  • Expose public domain qua Azure Front Door Premium.
  • Publish traffic vào app qua Gateway API layer.
  • Tích hợp Microsoft Entra External ID trực tiếp trong app.
  • Kiểm tra quyền theo path và slug ở tầng app.
  • Smoke check /reports/<slug>.
  • Notify Teams.

Khuyến nghị thực tế hiện tại

Với bối cảnh hiện tại, hướng nên chốt để đồng bộ nhất là AKS. Các giải pháp còn lại nên coi là phương án thay thế theo governance, chi phí hoặc mục đích riêng.
